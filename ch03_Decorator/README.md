# Decorator Pattern
装饰者模式：动态地将责任附加到对象上，若要扩展功能，装饰者提供了比继承更有弹性的替代方案。

装饰者和被装饰者必须是一样的类型。利用继承达到“类型匹配”，而不是利用继承获得“行为”。

新行为通过组合对象得到。
